# Examples

These examples are small, deterministic fixtures used for local development, CI smoke tests, and documentation.

- `portfolio`: basic portfolio/blog/docs workflow with stable filenames and common presets.
- `dogfood`: broader frontend-project workflow with content-hash filenames, cover and contain presets, helper export, and visual review artifact coverage.

Generated variants, manifests, reports, helper exports, and review artifacts are reproducible outputs. Regenerate them through `devimg`; do not hand-edit them.
