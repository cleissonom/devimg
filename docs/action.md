# GitHub Action

The Action in `action/` is a composite wrapper around the `devimg` CLI. It runs `check` or `optimize`, can optionally verify checked-in manifest helper files in check mode, and can generate a static review artifact for workflow upload.

```yaml
name: Image Pipeline

on:
  pull_request:
    paths:
      - "assets/images/**"
      - "devimg.toml"

jobs:
  images:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v6
      - uses: cleissonom/devimg/action@v0.1.11
        with:
          config: devimg.toml
          mode: check
          export-output: lib/devimg.generated.ts
          export-format: typescript
          strip-prefix: public
          url-prefix: /
          review-output: .devimg/review.html
      - uses: actions/upload-artifact@v4
        with:
          name: devimg-review
          path: .devimg/review.html
          if-no-files-found: error
```

Use `uses: ./action` plus `binary-path: target/debug/devimg` when testing the Action from this repository's local checkout after building the CLI. Consumer workflows should use the published Action path shown above.

`report-path` and `manifest-path` describe files for summary/output metadata. Configure the actual report and manifest paths in `devimg.toml`.

In `mode: check`, the Action runs `devimg check --no-report` so CI validation does not rewrite the configured Markdown report. The job summary uses the command output when no report file exists.

Set `export-output` when the project commits an app helper generated by `devimg manifest export`. In check mode, the Action runs `devimg check --no-report` first and then runs `devimg manifest export --check` against that file. The Action does not rewrite the helper file. `strip-prefix`, `url-prefix`, and `export-typescript-helpers` must match the command used to generate the checked-in export.

Set `fail-on-warning: true` only when unacknowledged advisory warnings should block CI. This includes `quality:` diagnostics for low lossy quality, upscaling, risky cover crops, and output-size surprises. Reviewed intentional warnings can be scoped in `[[warnings.acknowledge]]` and remain visible in reports.

Export inputs:

- `export-output`: checked-in helper/export file to verify.
- `export-manifest`: manifest path to export; defaults to `manifest-path`.
- `export-format`: `json` or `typescript`; defaults to `json`.
- `export-typescript-helpers`: set to `true` when the checked-in TypeScript export includes `--typescript-helpers`.
- `strip-prefix`: optional path prefix stripped from exported image URLs.
- `url-prefix`: optional URL prefix added to exported image URLs.

Set `review-output` when the workflow should produce a static visual review artifact. The Action runs `devimg review --manifest <manifest> --output <review-output>` after the main command and optional export check pass. `review-manifest` can override the manifest path used for the artifact; otherwise it defaults to `manifest-path`. `review-force: true` passes `--force` to replace an existing artifact. Pair this with `actions/upload-artifact` because the Action does not upload files automatically.

The default `version` input is `v0.1.11`, which maps to release assets such as `devimg-linux-x86_64.tar.gz`.

The MVP does not post PR comments, commit generated image files, rewrite reports in check mode, or rewrite manifest helper files.
