# GitHub Action

The Action in `action/` is a composite wrapper around the `devimg` CLI. It runs `check` or `optimize`, and in check mode it can optionally verify checked-in manifest helper files.

```yaml
name: Image Pipeline

on:
  pull_request:
    paths:
      - "assets/images/**"
      - "devimg.toml"

jobs:
  images:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v6
      - uses: cleissonom/devimg/action@v0.1.8
        with:
          config: devimg.toml
          mode: check
          export-output: lib/devimg.generated.ts
          export-format: typescript
          strip-prefix: public
          url-prefix: /
```

Use `uses: ./action` plus `binary-path: target/debug/devimg` when testing the Action from this repository's local checkout after building the CLI. Consumer workflows should use the published Action path shown above.

`report-path` and `manifest-path` describe files for summary/output metadata. Configure the actual report and manifest paths in `devimg.toml`.

Set `export-output` when the project commits an app helper generated by `devimg manifest export`. In check mode, the Action runs `devimg check` first and then runs `devimg manifest export --check` against that file. The Action does not rewrite the helper file. `strip-prefix` and `url-prefix` must match the command used to generate the checked-in export.

Export inputs:

- `export-output`: checked-in helper/export file to verify.
- `export-manifest`: manifest path to export; defaults to `manifest-path`.
- `export-format`: `json` or `typescript`; defaults to `json`.
- `strip-prefix`: optional path prefix stripped from exported image URLs.
- `url-prefix`: optional URL prefix added to exported image URLs.

The default `version` input is `v0.1.8`, which maps to release assets such as `devimg-linux-x86_64.tar.gz`.

The MVP does not post PR comments, commit generated image files, or rewrite manifest helper files.
